# Voice for cold outreach to AI/ML authors

The reader is a busy researcher whose inbox is full. They will judge whether
this is worth replying to in 5 seconds. Earn that 5 seconds.

## Do
- **Lead with proof you read the work.** Mention a specific design choice,
  result, or constraint from the paper / model card in the FIRST sentence.
- **Stay flat-hierarchy.** Write peer-to-peer, not student-to-celebrity. No
  "I'm a huge fan", no "your work is incredible", no "honored to reach out".
- **One concrete thing.** A single observation OR a single question — not three.
- **A low-friction ask.** "If you ever have a minute" / "if you're up for a
  short async exchange" / "no pressure to reply" — give them an easy out.
  An ask that costs them nothing is more likely to get yes.
- **Sign as a person.** Use placeholders `<your name>`, `<your background>`
  so the user fills them in. The user is a real person; this is not a bot blast.

## Don't
- "Hope this email finds you well" / "Hope you're doing great" — instant skip.
- "I'm reaching out because" / "I wanted to reach out" — meta-narration; just
  reach out.
- "Your work is fascinating" / "groundbreaking" / "incredible" — empty praise
  reads worse than no praise.
- Don't pitch. No "would love to share my project" without first earning interest.
- Don't ask for a 30-min call up front. Ask for a one-line reply.
- Don't include resume links, calendly links, or attachments.
- Don't BCC the user's manager. Just don't.

## Length

| Channel | Target length |
| --- | --- |
| Email | 3–5 short sentences. Body under 100 words. |
| Twitter / X DM | First message ≤ 280 chars; full content ≤ 1000 chars. |
| GitHub issue / discussion comment | 2–4 sentences; explicitly mark as a question, not a bug. |

## The "would I want to receive this?" test

Before finalizing, imagine receiving this exact message yourself with the
sender's name/work swapped for someone you don't know. Would you reply?

- "Yes, in 30 seconds." → ship.
- "Maybe, when I have time." → trim further.
- "Probably archive." → rewrite — the hook is too generic.
